异步通知

请求方式

POST请求，Content-Type为application/json

请求字段

字段类型示例说明是否签名
code整型100000请求后台返回操作状态，100000为成功操作否
msg字符串"OK"对code进行说明是
tradeNo字符串"1245648978477"商户单号是
orderNo字符串"998154152121156156"otc订单号是
paywayNo字符串"payway2020021321475023"payway订单号是
amount字符串"100"下单金额是
quantity字符串"14.6842878"资产数量是
receiveType字符串"UNIONPAY"用户付款方式(银行卡)是
userAccount字符串"6228480234564561654"用户银行卡/支付宝/微信账号是
userName字符串"张三"用户银行卡账户姓名是
userBankName字符串"中国银行"用户银行卡名称是
status字符串"完成"下单中,未付款,处理中,待退款,退款中,完成,失败是
merchantPayway字符串---币商收/付款方式是
merchantAccountNo字符串---币商方收付款账户是
merchantBankName字符串---币商开户行名称是
merchantName字符串---币商挂单名称是
remarks字符串---用户支付备注是
merchantBankAccountName字符串---商户方收付款账户昵称是
onlineUrl字符串http://**.com??order=1324564聊天链接,可以打开与商户一对一交流,完成订单是
timestamp字符串"1578992354262"时间戳是
type字符串"buy"订单方向buy(入金);sell(出金)是
finishedTime整型1578992354262订单完成时间,默认值为0否（该字段不签名）
sign字符串-签名否

注: 一、当对接的是聊天模式，返回信息中onlineUrl有请求地址时,请打开本链接,直接弹出对话框,与商家一对一交流,完成此币订单

二、请以返回订单信息中的"status"字段为订单状态，来确定流程的后续操作，"code"字段，仅仅表示为请求成功表示，不表示订单状态；

三、本次文档跟新，新增签名字段merchantName和remarks

返回内容

直接返回字符串

OK
