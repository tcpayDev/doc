异步通知

接入方请提供此接口, 订单完成后, tradingchain将会通过这个接口通知接入方订单状态

request:

参数名参数类型参数含义是否签名签名前是否加密(默认为否)
tradeNoString商户订单号Y-
orderNoString平台订单号Y-
amountString订单金额(原始下单金额)Y-
originString订单实际支付金额N该字段不参与签名
quantityStringusdt数量Y-
offerOrderNoString币商挂单单号Y-
typeString交易类型, buy:买, sell:卖Y-
userIdString用户idY-
statusString订单状态, 0:未付款, 1:待放币, 2:完成, 3:取消, 4:申述中Y-
msgStringY-
operSysTypeString终端操作系统类型, "ios" 或 "Android"Y-
rateString汇率, 钻石币对USDTY-
createTimeString订单创建时间Y-
endTimeString订单结束时间Y-
finishedTimelong订单完成时间N（该字段不签名）-
signString签名(secret与下单时的相同)--

请求示例

js: $.post({
    url: url,                     // 接入方申请接入时提供的url, 或者下单时的payNotifyUrl参数, 优先使用payNotifyUrl参数
    contentType: 'application/json; charset=UTF-8',
    data: JSON.stringify(data)    // data如下
})
