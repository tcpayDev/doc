Order status update

Our system provides two interfaces, pay notification and query. Pay notification will actively initiate order notifications to you. Only the server times out and return error code:500 will be resent. Other returns will not send notifications repeatedly.

To avoid not accepting pay notification, strongly recommended that you access the query interface.

  1. Access pay notification.

  2. Use the timing task to access the query interface.

    1. Query frequency suggestion：1min/10times.

    2. Query content suggestion：unfilled orders placed 10 mins ago.

Signature algorithm

  1. Sort the request parameters(excluding the sign parameter) according to the Ascii code sequence and splice them into string(during the splicing process, null values “”(empty) values and values that are not in string format are not required to be signed)

  2. Splice the apiSecret at the end

  3. Sha256 encrypt the concatenated string and convert to lower case.

Example
Need to request parameters and get apiSecret

apiSecret="abcdefg"

{
  "timestamp": 1583308458233,
  "id": "userId",
  "username": "username",
  "nickname": null,
  "age": ""
}
Splice request parameters

"id=userId&username=username"

Be care:

  • age is empty string，so don’t sign it.
  • nickname value is null，so don’t sign it.
  • timestamp value is not string, so don’t sign it
splice apiSecret

"id=userId&username=username&abcdefg"

Sha256 encrypt

"309403f3d530eaad5f716e88539fb7fea94b954643cd9bdb078938c27602e6ea"
