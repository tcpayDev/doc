模式1 - 标准入金

背景说明

原来下单过程是把所有请求参数拼成一个url, 直接在浏览器里打开完成的 假如有tradeNo(单号), amount(金额).. 等参数, 并且假如下单服务器地址是 ( http://api.tcpay.info/create ) 的话, 则拼接出 ( http://api.tcpay.info/create?tradeNo=xxx&amount=100& ) 这样的请求链接 请求过程直接是 玩家--> 接入方app--> 下单服务器 原接入方接口文档链接地址在此: ( https://github.com/Tradingchain/diamond-easy-mode/wiki )

而现在要做的是, 将请求过程调整为 玩家--> 接入方app--> 合作方服务器--> 下单服务器 因此封装了供合作方服务器调用的下单接口, 接口文档和请求返回示例如下
