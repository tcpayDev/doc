主动查询

请求方式

GET请求

请求URL

/mode1/find/orders

请求参数

字段类型示例说明是否签名
apiKey字符串"tc"商户请求标识否
orderLists字符串"%5B%22W-93409F8B88E344BD9E9FCFFA7812A0DF%22%2C%"商户订单号否

注

orderLists字段为List&lt<String&gt>，请求时需要将该字段URLEncoder.encode()或者JSON.toJSONString(),请传递转义之后的值;

该订单数量，一次性最大50条，大于50，则不会查询数据。

返回内容

字段类型示例说明
code整型100000请求后台返回操作状态，100000为成功操作
msg字符串"OK"对code进行说明
tradeNo字符串"1245648978477"商户单号
orderNo字符串"998154152121156156"平台方orderLists单号
paywayNo字符串"payway2020021321475023"payway订单号
amount字符串"100"下单金额
quantity字符串"14.6842878"资产数量
receiveType字符串"UNIONPAY"付款方式(银行卡)
userAccount字符串"6228480234564561654"银行卡/支付宝/微信账号
userName字符串"张三"银行卡账户姓名
userBankName字符串"中国银行"银行卡名称
status字符串"完成"下单中,未付款,处理中,待退款,退款中,完成,失败
merchantPayway字符串---平台方收/付款方式
merchantAccountNo字符串---平台方收付款账户
merchantBankAccountName字符串---平台方收付款账户昵称
merchantBankName字符串---平台方银行卡名称
onlineUrl字符串http://**.com??order=1324564聊天链接,可以打开与商户一对一交流,完成订单
finishedTime整型1578992354262订单完成时间,默认值为0
sign字符串-签名

注: 当对接的是聊天模式，则返回信息中onlineUrl有请求地址时,请打开本链接,直接弹出对话框,与商家一对一交流,完成此币订单；

请求示例：

URLEncoder.encode()： http://localhost:8080/mode1/find/orders?apiKey=tc-admin&orderLists=%5B%22W-93409F8B88E344BD9E9FCFFA7812A0DF%22%2C%22W-7E0B19356F7345DD884A23B25BE2D40E%22%2C%22W-7F8931A49D7741E1A92F577E92DB8A48%22%2C%22W-A78CC5BEBD454E128F83593DE9F0EB79%22%2C%22111%22%5D

JSON.toJSONString() http://localhost:8080/mode1/find/orders?apiKey=tc-admin&orderLists=["W-93409F8B88E344BD9E9FCFFA7812A0DF","W-7E0B19356F7345DD884A23B25BE2D40E","W-7F8931A49D7741E1A92F577E92DB8A48","W-A78CC5BEBD454E128F83593DE9F0EB79","111"]
