准备工作

您已经拿到了apiKey和apiSecret

订单状态更新

本系统提供了异步通知和主动查询两个接口。异步通知会在您的系统返回OK后停止发送，否则，系统将在一个小时内重复发送共8次。

也就是说，您需要在您接收异步通知的时候，可能会受到重复的订单消息。

为了避免没有接收到异步通知的问题，强烈建议您接入主动查询的接口。

最佳实践

  1. 接入异步通知。
  2. 使用定时任务的方式接入主动查询接口。
    1. 查询频次建议：5分钟/次。
    2. 查询内容建议：10分钟前下单的未完成订单。

签名算法

  1. 将请求参数(排除sign参数)按Ascii码序进行排序并拼接成字符串(拼接过程中,null值""(空)值以及不是string格式的值都不用进行签名)。
  2. 将发放的apiSecret拼接在最后。
  3. 将拼接后的字符串进行sha256加密, 转成小写。
示例
需要请求的参数和得到的apiSecret

apiSecret="abcdefg"

{
  "timestamp": 1583308458233,
  "id": "userId",
  "username": "username",
  "nickname": null,
  "age": ""
}
拼接请求参数

"id=userId&username=username"

注意：

  • age为空字符串，所以不参加签名
  • nickname值为空，所以不参加签名
  • timestamp的值不是string,所以不参加签名
拼接apiSecret

"id=userId&username=username&abcdefg"

进行SHA256加密
